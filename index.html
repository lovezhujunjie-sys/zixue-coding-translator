第一站 · 建立感觉

代码,其实就是"给电脑写的菜谱"

电脑很笨,它什么都不会主动做。你得把每一步都写清楚,它才会照做。这份"步骤说明书"就是代码。看懂代码 = 看懂这份说明书的每一句话。

1代码 = 一条条命令

程序员把任务拆成很多小步骤,每一行代码通常就是一个命令,告诉电脑"做这件事"。电脑从上往下,一行一行地执行,绝不会跳着读。

🍜 打个比方:就像泡面说明——「①撕开包装 ②加调料 ③倒开水 ④等3分钟」。电脑就是那个严格按顺序、绝不偷懒也绝不动脑的厨师。你写错一步,它就照着错的做。

2变量 = 贴了标签的盒子

代码里经常出现 name = "小明" 这种东西。意思是:准备一个盒子,贴上"name"的标签,把"小明"放进去。以后说 name,就是指里面的"小明"。盒子里的东西随时能换成别的。

📦 打个比方:就像你在冰箱上贴便利贴。"鸡蛋在第二层"——下次你不用翻冰箱,看标签就知道鸡蛋在哪。把鸡蛋吃完换成牛奶,标签内容也跟着变。

3数据类型 = 盒子里装的"东西种类"

盒子里能装不同种类的东西,代码主要分这几类:数字(18、3.5)能做算术;文字(要用引号包起来,像 "你好")是一串字符;真/假(True / False)用来回答"是不是"。

🏷️ 打个比方:18 是数字,能加减乘除;"18" 加了引号就变成文字了,电脑会把它当成两个字符,而不是十八。这个区别超级重要,新手最容易在这栽跟头。

4函数 = 一个能重复用的"按钮"

有些事要反复做,程序员就把它打包成一个"函数",起个名字。以后想做这件事,喊一声名字加个括号就行(像 print()),不用重写一遍。括号里塞的是"给它的材料"。

☕ 打个比方:就像咖啡机上的"美式"按钮。背后有一堆复杂步骤(磨豆、加水、加热),但你只要按一下,它全帮你做完。括号里放的糖和奶,就是你给它的"材料"。

5判断(if)= 代码会"看情况办事"

电脑能根据情况走不同的路。if(如果)后面跟一个条件,条件成立就做一件事,不成立就跳过或改做别的(else,否则)。这是代码"聪明"起来的地方。

☔ 打个比方:"如果下雨,就带伞;否则,穿短袖。"你每天出门都在做这种判断,代码也一样。条件就是那句"下雨了吗?"

6循环(for / while)= 重复做,不喊累

要做同一件事很多遍?不用复制粘贴一百行。for 和 while 让电脑自动重复,做几遍你说了算。这是电脑最擅长、人最讨厌的活。

🏃 打个比方:体育老师说"绕操场跑5圈"。他不会喊5遍"跑一圈",而是说一句"跑5圈"。循环就是那句"重复N遍"的指令。

7列表 = 一排格子的盒子

普通盒子只能装一样东西,列表(用方括号 [ ] 表示)能一次装一排,像 ["苹果", "香蕉", "橘子"]。要装一堆同类东西时就用它,比如一个班所有同学的名字。

🥚 打个比方:就像鸡蛋托盘,一格一个,排得整整齐齐。你可以说"给我第1个""给我第3个",每个格子都有自己的编号(而且程序员从 0 开始数!)。

8输入输出 = 电脑怎么跟你"对话"

输出(print)是电脑"说话"给你看;输入(input)是电脑"提问"等你回答。一个程序能跟人互动,靠的就是这一进一出。

💬 打个比方:就像微信聊天。print 是对方发来的消息(显示给你看);input 是输入框,等你打字回复。程序就是这样一来一回跟你聊起来的。

9看懂代码的"三板斧"

遇到任何一段代码,先别慌。你只要从上往下,逐行问自己三个问题,就能懂个七八成:

🪓 ① 它在存什么? 找等号 = ,看哪个盒子装了什么东西
🪓 ② 它在做什么? 找带括号的词(如 print),那通常是一个动作
🪓 ③ 它在判断或重复什么? 找 if(如果)、for(重复)这种词,看代码在什么情况下拐弯、重复几遍

练熟这三板斧,90% 的入门代码你都能读个明白。下一站就用它来实战!

🎯 写代码的小心法

1. 引号别漏。文字内容一定要用 " " 包起来,比如 "你好"。漏了电脑会看不懂。

2. 缩进很重要。if 下面那行要往右空几格(按一下 Tab 键),表示"这行属于它"。

3. 出错很正常。哪怕是十年的老程序员,代码也经常一次跑不过。报错不是骂你,是电脑在提示哪里没对。改了再试就好。
